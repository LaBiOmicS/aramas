# Contributing to ARAMAS

First off, thank you for considering contributing to ARAMAS! It's people like you that make ARAMAS such a great tool for the community.

## How Can I Contribute?

### Reporting Bugs
* Check the [issue tracker](https://github.com/LaBiOmicS/aramas/issues) to see if the bug has already been reported.
* If you can't find an open issue addressing the problem, [open a new one](https://github.com/LaBiOmicS/aramas/issues/new). Be as specific as possible! Include your operating system, browser version, and steps to reproduce the bug.

### Suggesting Enhancements
* Enhancement suggestions are tracked as [GitHub issues](https://github.com/LaBiOmicS/aramas/issues).
* Open a new issue and describe the feature you'd like to see, why it's useful, and how it should work.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints (`npm run lint`).
6. Issue that pull request!

## Styleguide

### JavaScript/TypeScript Styleguide
* We use ESLint for linting. Please ensure your code follows the project's rules.
* Use meaningful variable and function names.
* Comment your code where necessary, especially for complex logic.

### Commit Messages
* Use the present tense ("Add feature" not "Added feature").
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
* Limit the first line to 72 characters or less.
* Reference issues and pull requests liberally after the first line.
